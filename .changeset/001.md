### `useInkwell` hook

Adds the `useInkwell(options)` hook as the recommended editor API. Returns
`{ state, EditorInstance, editor }`, where `editor` is a stable controller
exposing imperative actions (`focus`, `clear`, `setContent`, `insertContent`,
and `getState`).

### Default stylesheet

Ships a default editor, plugin, and renderer stylesheet at
`@railway/inkwell/styles.css`. Designed as a neutral, theme-aware baseline —
light by default, dark via `prefers-color-scheme: dark` — wired up via CSS
custom properties (`--inkwell-bg`, `--inkwell-text`, `--inkwell-border`,
`--inkwell-accent`, `--inkwell-code-bg`, etc.) so consumers can re-theme
without rewriting selectors.

### New built-in plugins

- `createEmojiPlugin({ emojis?, search?, renderItem?, trigger?, emptyMessage? })`
  — searchable `:` emoji picker that opens at token boundaries, ships with a
  default emoji set, and exposes hooks for custom/async emoji sources.
- `createMentionsPlugin({ name, trigger, marker, search, renderItem, onSelect?, emptyMessage? })`
  — generic trigger-based searchable picker. Inserts either `@<marker>[<id>]`
  by default or the string returned by `onSelect`. Renderer-side hydration via
  the new `MentionRenderer` type and `mentions` prop on `<InkwellRenderer />`.
- `createAttachmentsPlugin({ accept, onUpload, onError?, uploadingPlaceholder? })`
  — paste or drop image files into the editor; also handles copied HTML
  `<img>` tags by inserting each safe `src` URL directly as an image block
  without calling `onUpload`.
- `createSlashCommandsPlugin({ commands, onReadyChange?, onExecute?, name?, emptyMessage? })`
  — reusable Discord-style slash-command autocomplete for chat-style
  composers. Activates from `/` typed with no prose before the cursor on
  the current line, claims editor activation imperatively via
  `ctx.setActivePlugin` (no character trigger), and writes the slash
  line directly through the editor (no content round-trip required from the host). Supports command filtering from
  typed editor text, disabled commands, required first-argument
  choices, async choice loading, completion readiness, keyboard
  navigation, cursor-positioned rendering, and renders through the
  shared `.inkwell-plugin-picker-*` menu classes. `onExecute` receives a
  string-only structured payload (`{ name, args, raw }`). Executing or
  canceling from the ready phase clears only the slash-command line.
- `createCompletionsPlugin({ getCompletion, isLoading?, loadingText?, acceptHint?, onAccept?, onDismiss?, onRestore? })`
  — generic placeholder completions for suggested text flows. Shows the current
  completion through the editor's native placeholder while the document is
  empty, prefixes it with `acceptHint` (default `[tab ↹]`), accepts with Tab,
  dismisses with Escape or normal typing, and can restore the accepted
  completion when undo returns the document to empty. Uses the new
  `onEditorChange` plugin hook instead of `setup`, so plugin objects can be
  created inline or with normal React dependencies without stale closure refs.
  While active, an otherwise empty editor is normalized to a single plain
  paragraph so placeholder typography stays stable after clears.

### Shared plugin picker primitive

Snippets, mentions, and emoji all render through a shared
`PluginMenuPrimitive` so the menu UI, editor-driven query display,
keyboard navigation, scoped key-forwarding (via the new
`subscribeForwardedKey` callback on `PluginRenderProps`), and focus
behavior are identical across plugins. The channel is scoped per editor
instance — two editors on the same page do not cross-talk.

**Breaking (CSS).** The previous plugin-specific class namespaces
`.inkwell-plugin-snippets-*` and `.inkwell-plugin-mentions-*` are replaced by
the shared `.inkwell-plugin-picker-*` namespace:

| Old                                      | New                                |
| ---------------------------------------- | ---------------------------------- |
| `.inkwell-plugin-snippets-popup`         | `.inkwell-plugin-picker-popup`     |
| `.inkwell-plugin-snippets-picker`        | `.inkwell-plugin-picker`           |
| `.inkwell-plugin-snippets-search`        | `.inkwell-plugin-picker-search`    |
| `.inkwell-plugin-snippets-item`          | `.inkwell-plugin-picker-item`      |
| `.inkwell-plugin-snippets-item-active`   | `.inkwell-plugin-picker-item-active` |
| `.inkwell-plugin-snippets-title`         | `.inkwell-plugin-picker-title`     |
| `.inkwell-plugin-snippets-preview`       | `.inkwell-plugin-picker-preview`   |
| `.inkwell-plugin-snippets-empty`         | `.inkwell-plugin-picker-empty`     |
| `.inkwell-plugin-mentions-*` (any)       | `.inkwell-plugin-picker-*`         |

`.inkwell-plugin-picker-subtitle` is new. Custom plugins built on top of
`PluginMenuPrimitive` automatically inherit the same class namespace.

### Character-limit additions

- `characterLimit?: number`, `enforceCharacterLimit?: boolean`,
  `onCharacterCount?: (count, limit?) => void` props (also available via
  `useInkwell`).
- New `limitToast?: boolean` prop (default `true`) renders a built-in toast at
  the top-right of `.inkwell-editor-wrapper` when `characterCount >
  characterLimit`. With `enforceCharacterLimit`, it also renders at exactly the
  limit because further typing is blocked. Styled via
  `.inkwell-editor-limit-toast`. Pass `limitToast={false}` to opt out and
  render your own indicator.
- `.inkwell-editor-over-limit` is added to the wrapper whenever
  `characterCount > characterLimit`.

### Editor state + lifecycle

- `editable?: boolean` (default `true`) for read-only mode.
- `style?: React.CSSProperties` applies inline styles to the editable surface,
  useful for sizing the editor with `height`, `minHeight`, or `width`.
- `submitOnEnter?: boolean` and `onSubmit?: (content) => void` for chat-style
  composers where Enter submits and Shift+Enter inserts a newline.
- `onStateChange?: (state: InkwellEditorState) => void` reporting
  `{ content, isEmpty, isFocused, isEditable, characterCount, characterLimit, overLimit }`.
- Plugins may opt into a `setup(editor)` lifecycle hook.
- New `activatable?: boolean` plugin flag for plugins (like slash
  commands) that have no character `trigger` but still want to be
  rendered only while they are the editor's active plugin.
- New `setActivePlugin` callback on `PluginKeyDownContext`, so plugins
  can claim or release editor activation imperatively from their
  `onKeyDown` handler.
- New `subscribeForwardedKey` callback on `PluginRenderProps` exposing
  the per-plugin, per-editor key-forwarding channel.

### Markdown image and renderer additions

- Editor image block parsing, rendering, and serialization round-trip for
  standalone `![alt](url)` lines.
- `mentions: MentionRenderer[]` prop on `<InkwellRenderer />` to hydrate
  Markdown markers into custom React nodes.

### Better paste handling

The attachments plugin handles copied HTML `<img>` tags by inserting each safe
`src` URL directly as an image block without calling `onUpload`; unsafe or
missing `src` values are ignored. File paste/drop continues to use `onUpload`.
Slate's Markdown paste pipeline also recognizes standalone `![alt](url)` image
lines.
