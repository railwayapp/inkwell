Inkwell is a WYSIWYG Markdown editor for React. The content you see is
visually formatted, but the underlying data is always a plain Markdown
string.

## Usage

The recommended API is `useInkwell`. Give it your Markdown string and an
`onChange` handler, render the returned `EditorInstance`, and use the
grouped `editor` controller when you need to inspect or manipulate the
document.

```tsx
import { useInkwell } from "@railway/inkwell";
import { useState } from "react";

function App() {
  const [content, setContent] = useState("# Hello **world**");
  const { state, EditorInstance, editor } = useInkwell({
    content,
    onChange: setContent,
    placeholder: "Write something...",
  });

  return (
    <>
      <EditorInstance />
      <p>{state.characterCount} characters</p>
      <button onClick={() => editor.focus({ at: "end" })}>Focus editor</button>
      <button onClick={() => editor.clear()}>Clear</button>
    </>
  );
}
```

`content` accepts any Markdown string. `onChange` fires on every edit with
the updated Markdown.

## Low-level component

Most applications should use `useInkwell`. The lower-level `InkwellEditor`
component is exported for custom abstractions and integrations that need to
own component rendering directly. Prefer the hook unless you have a specific
reason not to use it.

```tsx
import { InkwellEditor } from "@railway/inkwell";

<InkwellEditor content={content} onChange={setContent} />;
```

## Editor controller

The `editor` object returned by `useInkwell` exposes `getMarkdown()`,
`getText()`, `getState()`, `focus()`, `clear()`, `setMarkdown()`, and
`insertMarkdown()`.

```tsx
editor.getMarkdown();
editor.getText();
editor.getState();
editor.focus({ at: "end" });
editor.setMarkdown("Replacement", { emitChange: false });
editor.insertMarkdown(" **more**");
```

If you are building a custom abstraction around the low-level component,
the same methods are available through
`ref={useRef<InkwellEditorHandle>(null)}` on `InkwellEditor`.

## Hook return value

`useInkwell(options)` returns:

- `state` — the latest `InkwellEditorState` snapshot.
- `EditorInstance` — a stable React component that renders the configured
  editor. Render it once per hook call.
- `editor` — the grouped controller for reading and manipulating content.

## Formatting

Type Markdown syntax and the editor renders it visually as you go. Type
`## ` at the start of a line and it becomes a heading. Wrap text in `**`
and it turns bold. The Markdown characters stay in the document — they're
styled to show you what the output will look like.

All formatting is enabled by default.

### Block formatting

Type these at the beginning of a line:

| Pattern                | Result             |
| ---------------------- | ------------------ |
| `# ` through `###### ` | Headings (h1–h6)   |
| `- `, `* `, or `+ `    | List items         |
| `> `                   | Blockquotes        |
| ` ``` `                | Fenced code blocks |

### Inline formatting

Wrap text with these markers anywhere in a line:

| Syntax       | Result            |
| ------------ | ----------------- |
| `**text**`   | **Bold**          |
| `_text_`     | _Italic_          |
| `~~text~~`   | ~~Strikethrough~~ |
| `` `text` `` | `Inline code`     |

### Keyboard shortcuts

| Shortcut        | Action               |
| --------------- | -------------------- |
| `⌘B` / `Ctrl+B` | Toggle bold          |
| `⌘I` / `Ctrl+I` | Toggle italic        |
| `⌘D` / `Ctrl+D` | Toggle strikethrough |

These shortcuts come from the built-in
[bubble menu plugin](/docs/editor-plugins#bubble-menu). You can customize
or disable them.

## Disabling formatting

Use the `decorations` option to turn off specific block-level formatting.
All decorations are enabled by default — you only need this prop to
disable something.

```tsx
const { EditorInstance } = useInkwell({
  content,
  onChange: setContent,
  decorations: {
    codeBlocks: false,
    heading4: false,
    heading5: false,
    heading6: false,
  },
});

return <EditorInstance />;
```

| Decoration              | Default | Controls                             |
| ----------------------- | ------- | ------------------------------------ |
| `heading1` – `heading6` | `true`  | Individual heading levels            |
| `lists`                 | `true`  | Unordered list items (`-`, `*`, `+`) |
| `blockquotes`           | `true`  | Blockquotes (`>`)                    |
| `codeBlocks`            | `true`  | Fenced code blocks (` ``` `)         |

When a decoration is disabled, its syntax is treated as plain text with
no visual formatting.

## Syntax highlighting

Fenced code blocks are highlighted with
[highlight.js](https://highlightjs.org/) by default. Import a highlight.js
CSS theme for colors to appear:

```tsx
import "highlight.js/styles/github-dark.css";
```

To use a different highlighter, pass it through the `rehypePlugins` option:

```tsx
import { useInkwell } from "@railway/inkwell";
import rehypeShiki from "@shikijs/rehype";

const { EditorInstance } = useInkwell({
  content,
  onChange: setContent,
  rehypePlugins: [[rehypeShiki, { theme: "github-dark" }]],
});

return <EditorInstance />;
```

The same `rehypePlugins` option is available on
[`InkwellRenderer`](/docs/renderer).

## Bubble menu

The editor includes a floating formatting toolbar by default. Select text
and the toolbar appears with bold, italic, and strikethrough buttons.

To disable it:

```tsx
const { EditorInstance } = useInkwell({
  content,
  onChange: setContent,
  bubbleMenu: false,
});

return <EditorInstance />;
```

To customize the toolbar items, see the
[Plugins](/docs/editor-plugins#bubble-menu) guide.

## Options reference

### `content`

**Type:** `string`

The Markdown string to display and edit. When
[collaboration](/docs/collaboration) is enabled, this only seeds an empty
document — the Yjs shared type becomes the source of truth after that.

### `onChange`

**Type:** `(markdown: string) => void`

Called on every document change with the updated Markdown string.

### `onStateChange`

**Type:** `(state: InkwellEditorState) => void`

Called with a full state snapshot when content, focus, editability, or
character count changes. The state includes `markdown`, `text`, `isEmpty`,
`isFocused`, `isEditable`, `characterCount`, `characterLimit`, and
`overLimit`.

### `placeholder`

**Type:** `string`

Text shown when the editor is empty.

### `className`

**Type:** `string`

CSS class applied to the outer wrapper element.

### `editable`

**Type:** `boolean` — default: `true`

Whether users can edit the document. Pass `false` for read-only composer
states.

### `decorations`

**Type:** `InkwellDecorations`

Controls which block-level formatting the editor recognizes. All enabled
by default. See [Disabling formatting](#disabling-formatting).

### `plugins`

**Type:** `InkwellPlugin[]`

Additional plugins to load. See [Plugins](/docs/editor-plugins).

### `bubbleMenu`

**Type:** `boolean` — default: `true`

Whether to include the built-in floating toolbar. Pass `false` to disable
it entirely, or use a custom bubble menu via `plugins` instead.

### `collaboration`

**Type:** `CollaborationConfig`

Enable real-time collaborative editing. See
[Collaboration](/docs/collaboration).

### `rehypePlugins`

**Type:** `RehypePluginConfig[]`

Custom rehype plugins for code block highlighting. Accepts a plugin
function or a `[plugin, options]` tuple. See
[Syntax highlighting](#syntax-highlighting).

### `characterLimit`

**Type:** `number`

Tracks the current character count and adds `inkwell-editor-over-limit` to
the wrapper when exceeded. When set, the editor also renders a built-in
[limit toast](#limittoast) at the top-right unless disabled.

### `enforceCharacterLimit`

**Type:** `boolean` — default: `false`

When `true`, blocks text input past `characterLimit`.

### `onCharacterCount`

**Type:** `(count: number, limit?: number) => void`

Called whenever the editor recalculates character count.

### `submitOnEnter`

**Type:** `boolean` — default: `false`

When `true`, pressing Enter calls `onSubmit(markdown)` instead of inserting a
new line. Shift+Enter still inserts a line break. Useful for chat-style
composers.

### `onSubmit`

**Type:** `(markdown: string) => void`

Called when `submitOnEnter` handles Enter.

### `limitToast`

**Type:** `boolean` — default: `true`

When a `characterLimit` is configured, the editor renders a small toast at
the top-right of `.inkwell-editor-wrapper` whenever the document hits the
limit. Visibility:

- `characterCount > characterLimit` → shows `"Over limit by N"`.
- `enforceCharacterLimit && characterCount >= characterLimit` → shows
  `"Character limit reached"` so users hitting the wall under enforce mode
  still get feedback.
- Exactly at the limit but not enforced → hidden.

The toast is `pointer-events: none` so it never blocks typing or selection,
and uses `role="status"` + `aria-live="polite"` for screen readers. Style
it via `.inkwell-editor-limit-toast` (and
`.inkwell-editor-limit-toast-icon`) in your stylesheet.

Pass `limitToast={false}` to opt out and render your own indicator via
`onCharacterCount`.
